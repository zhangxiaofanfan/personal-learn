问题1: 自动注入功能出现问题: 当组件需要注入的依赖已经存在在容器中则可以正常依赖注入, 但是如果需要依赖注入的组件还没有被注册到容器中则会报错;
问题2: 使用绝对路径查找 ComponentScan 扫描包下的类文件;
    解决方案: 在第二版本可以使用相对路径对扫描 Component 组件;
问题3: public <T> T getBean(String beanName, Class<T> beanClass): 方法规则不友好;
    解决方案: 第二版 Bean 对象获取规则:
        1. 按照 Class 类型获取, 如果该类型的对象在容器中只存在一个, 则将这个对象进行返回;
        2. 如果该类型的对象存在多个, 按照名称获取, 返回查找到第一个 Bean 对象;
        3. 按照类型查找不到, 或者查找到多个, 按照 BeanName 查找不到, 抛出异常并退出;
